    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#d3-path" aria-hidden="true" class="anchor" id="user-content-d3-path"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>d3-path</h1>
<p>Say you have some code that draws to a 2D canvas:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">function</span> <span class="pl-en">drawCircle</span>(<span class="pl-smi">context</span>, <span class="pl-smi">radius</span>) {
  <span class="pl-smi">context</span>.<span class="pl-c1">moveTo</span>(radius, <span class="pl-c1">0</span>);
  <span class="pl-smi">context</span>.<span class="pl-en">arc</span>(<span class="pl-c1">0</span>, <span class="pl-c1">0</span>, radius, <span class="pl-c1">0</span>, <span class="pl-c1">2</span> <span class="pl-k">*</span> <span class="pl-c1">Math</span>.<span class="pl-c1">PI</span>);
}</pre></div>
<p>The d3-path module lets you take this exact code and additionally render to <a href="http://www.w3.org/TR/SVG/paths.html">SVG</a>. It works by <a href="#path_toString">serializing</a> <a href="http://www.w3.org/TR/2dcontext/#canvaspathmethods">CanvasPathMethods</a> calls to <a href="http://www.w3.org/TR/SVG/paths.html#PathData">SVG path data</a>. For example:</p>
<div class="highlight highlight-source-js"><pre><span class="pl-k">var</span> context <span class="pl-k">=</span> <span class="pl-smi">d3</span>.<span class="pl-en">path</span>();
<span class="pl-en">drawCircle</span>(context, <span class="pl-c1">40</span>);
<span class="pl-smi">pathElement</span>.<span class="pl-c1">setAttribute</span>(<span class="pl-s"><span class="pl-pds">"</span>d<span class="pl-pds">"</span></span>, <span class="pl-smi">context</span>.<span class="pl-c1">toString</span>());</pre></div>
<p>Now code you write once can be used with both Canvas (for performance) and SVG (for convenience). For a practical example, see <a href="https://github.com/d3/d3-shape">d3-shape</a>.</p>
<h2><a href="#installing" aria-hidden="true" class="anchor" id="user-content-installing"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installing</h2>
<p>If you use NPM, <code>npm install d3-path</code>. Otherwise, download the <a href="https://github.com/d3/d3-path/releases/latest">latest release</a>. You can also load directly from <a href="https://d3js.org">d3js.org</a>, either as a <a href="https://d3js.org/d3-path.v1.min.js">standalone library</a> or as part of <a href="https://github.com/d3/d3">D3 4.0</a>. AMD, CommonJS, and vanilla environments are supported. In vanilla, a <code>d3</code> global is exported:</p>
<div class="highlight highlight-text-html-basic"><pre>&lt;<span class="pl-ent">script</span> <span class="pl-e">src</span>=<span class="pl-s"><span class="pl-pds">"</span>https://d3js.org/d3-path.v1.min.js<span class="pl-pds">"</span></span>&gt;&lt;/<span class="pl-ent">script</span>&gt;
&lt;<span class="pl-ent">script</span>&gt;<span class="pl-s1"></span>
<span class="pl-s1"></span>
<span class="pl-s1"><span class="pl-k">var</span> path <span class="pl-k">=</span> <span class="pl-smi">d3</span>.<span class="pl-en">path</span>();</span>
<span class="pl-s1"><span class="pl-smi">path</span>.<span class="pl-c1">moveTo</span>(<span class="pl-c1">1</span>, <span class="pl-c1">2</span>);</span>
<span class="pl-s1"><span class="pl-smi">path</span>.<span class="pl-en">lineTo</span>(<span class="pl-c1">3</span>, <span class="pl-c1">4</span>);</span>
<span class="pl-s1"><span class="pl-smi">path</span>.<span class="pl-en">closePath</span>();</span>
<span class="pl-s1"></span>
<span class="pl-s1"></span><span class="pl-s1">&lt;</span>/<span class="pl-ent">script</span>&gt;</pre></div>
<p><a href="https://tonicdev.com/npm/d3-path">Try d3-path in your browser.</a></p>
<h2><a href="#api-reference" aria-hidden="true" class="anchor" id="user-content-api-reference"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>API Reference</h2>
<p><a name="user-content-path" href="#path">#</a> d3.<b>path</b>() <a href="https://github.com/d3/d3-path/blob/master/src/path.js" title="Source">&lt;&gt;</a></p>
<p>Constructs a new path serializer that implements <a href="http://www.w3.org/TR/2dcontext/#canvaspathmethods">CanvasPathMethods</a>.</p>
<p><a name="user-content-path_moveTo" href="#path_moveTo">#</a> <i>path</i>.<b>moveTo</b>(<i>x</i>, <i>y</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L18" title="Source">&lt;&gt;</a></p>
<p>Move to the specified point ⟨<em>x</em>, <em>y</em>⟩. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-moveto"><em>context</em>.moveTo</a> and SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataMovetoCommands">“moveto” command</a>.</p>
<p><a name="user-content-path_closePath" href="#path_closePath">#</a> <i>path</i>.<b>closePath</b>() <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L21" title="Source">&lt;&gt;</a></p>
<p>Ends the current subpath and causes an automatic straight line to be drawn from the current point to the initial point of the current subpath. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-closepath"><em>context</em>.closePath</a> and SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataClosePathCommand">“closepath” command</a>.</p>
<p><a name="user-content-path_lineTo" href="#path_lineTo">#</a> <i>path</i>.<b>lineTo</b>(<i>x</i>, <i>y</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L27" title="Source">&lt;&gt;</a></p>
<p>Draws a straight line from the current point to the specified point ⟨<em>x</em>, <em>y</em>⟩. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-lineto"><em>context</em>.lineTo</a> and SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataLinetoCommands">“lineto” command</a>.</p>
<p><a name="user-content-path_quadraticCurveTo" href="#path_quadraticCurveTo">#</a> <i>path</i>.<b>quadraticCurveTo</b>(<i>cpx</i>, <i>cpy</i>, <i>x</i>, <i>y</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L30" title="Source">&lt;&gt;</a></p>
<p>Draws a quadratic Bézier segment from the current point to the specified point ⟨<em>x</em>, <em>y</em>⟩, with the specified control point ⟨<em>cpx</em>, <em>cpy</em>⟩. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-quadraticcurveto"><em>context</em>.quadraticCurveTo</a> and SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataQuadraticBezierCommands">quadratic Bézier curve commands</a>.</p>
<p><a name="user-content-path_bezierCurveTo" href="#path_bezierCurveTo">#</a> <i>path</i>.<b>bezierCurveTo</b>(<i>cpx1</i>, <i>cpy1</i>, <i>cpx2</i>, <i>cpy2</i>, <i>x</i>, <i>y</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L33" title="Source">&lt;&gt;</a></p>
<p>Draws a cubic Bézier segment from the current point to the specified point ⟨<em>x</em>, <em>y</em>⟩, with the specified control points ⟨<em>cpx1</em>, <em>cpy1</em>⟩ and ⟨<em>cpx2</em>, <em>cpy2</em>⟩. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-beziercurveto"><em>context</em>.bezierCurveTo</a> and SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataCubicBezierCommands">cubic Bézier curve commands</a>.</p>
<p><a name="user-content-path_arcTo" href="#path_arcTo">#</a> <i>path</i>.<b>arcTo</b>(<i>x1</i>, <i>y1</i>, <i>x2</i>, <i>y2</i>, <i>radius</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L36" title="Source">&lt;&gt;</a></p>
<p>Draws a circular arc segment with the specified <em>radius</em> that starts tangent to the line between the current point and the specified point ⟨<em>x1</em>, <em>y1</em>⟩ and ends tangent to the line between the specified points ⟨<em>x1</em>, <em>y1</em>⟩ and ⟨<em>x2</em>, <em>y2</em>⟩. If the first tangent point is not equal to the current point, a straight line is drawn between the current point and the first tangent point. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-arcto"><em>context</em>.arcTo</a> and uses SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataEllipticalArcCommands">elliptical arc curve commands</a>.</p>
<p><a name="user-content-path_arc" href="#path_arc">#</a> <i>path</i>.<b>arc</b>(<i>x</i>, <i>y</i>, <i>radius</i>, <i>startAngle</i>, <i>endAngle</i>[, <i>anticlockwise</i>]) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L84" title="Source">&lt;&gt;</a></p>
<p>Draws a circular arc segment with the specified center ⟨<em>x</em>, <em>y</em>⟩, <em>radius</em>, <em>startAngle</em> and <em>endAngle</em>. If <em>anticlockwise</em> is true, the arc is drawn in the anticlockwise direction; otherwise, it is drawn in the clockwise direction. If the current point is not equal to the starting point of the arc, a straight line is drawn from the current point to the start of the arc. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-arc"><em>context</em>.arc</a> and uses SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataEllipticalArcCommands">elliptical arc curve commands</a>.</p>
<p><a name="user-content-path_rect" href="#path_rect">#</a> <i>path</i>.<b>rect</b>(<i>x</i>, <i>y</i>, <i>w</i>, <i>h</i>) <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L122" title="Source">&lt;&gt;</a></p>
<p>Creates a new subpath containing just the four points ⟨<em>x</em>, <em>y</em>⟩, ⟨<em>x</em> + <em>w</em>, <em>y</em>⟩, ⟨<em>x</em> + <em>w</em>, <em>y</em> + <em>h</em>⟩, ⟨<em>x</em>, <em>y</em> + <em>h</em>⟩, with those four points connected by straight lines, and then marks the subpath as closed. Equivalent to <a href="http://www.w3.org/TR/2dcontext/#dom-context-2d-rect"><em>context</em>.rect</a> and uses SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathDataLinetoCommands">“lineto” commands</a>.</p>
<p><a name="user-content-path_toString" href="#path_toString">#</a> <i>path</i>.<b>toString</b>() <a href="https://github.com/d3/d3-path/blob/master/src/path.js#L125" title="Source">&lt;&gt;</a></p>
<p>Returns the string representation of this <em>path</em> according to SVG’s <a href="http://www.w3.org/TR/SVG/paths.html#PathData">path data specficiation</a>.</p>
